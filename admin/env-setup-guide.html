🔧 EdgeOne环境变量配置指南

⚠️ 重要提示
您的诊断显示401 "Invalid credentials"错误，这表明EdgeOne环境变量未正确配置。请按以下步骤操作：

📋 第一步：进入EdgeOne控制台

  1. 登录 EdgeOne Pages控制台
  2. 找到您的项目：company
  3. 点击进入项目详情页

⚙️ 第二步：配置环境变量

  1. 在项目页面中找到 "设置" 或 "Settings" 选项
  2. 点击 "环境变量" 或 "Environment Variables"
  3. 添加以下三个环境变量：

🔑 必需的环境变量

变量名称: ADMIN_USERNAME
变量值: 您的管理员用户名
说明: 用于登录管理后台的用户名
变量名称: ADMIN_PASSWORD
变量值: 您的管理员密码
说明: 用于登录管理后台的密码
变量名称: GITEE_TOKEN
变量值: 48eb317592d7a716e5cdc03411e51d37
说明: Gitee访问令牌，用于同步配置到仓库
🔒 安全注意事项：
• 用户名和密码请设置为强密码
• 请妥善保管这些凭据
• 不要在代码中硬编码这些信息

🚀 第三步：重新部署

  1. 保存环境变量配置
  2. 在EdgeOne Pages中点击 "重新部署" 或 "Redeploy"
  3. 等待部署完成（通常需要1-3分钟）

❓ 常见问题解决

Q: 配置后仍然显示401错误？

解决方案：
  1. 确认环境变量名称完全匹配（区分大小写）
  2. 检查是否有多余的空格
  3. 重新部署项目
  4. 清除浏览器缓存后重试

Q: 找不到环境变量配置页面？

解决方案：
  1. 确认您有项目的管理权限
  2. 查看项目设置中的"Functions"或"变量"选项
  3. 联系EdgeOne技术支持
🎉 配置成功后：
• 可以正常登录管理后台
• 支持在线编辑网站内容
• 自动同步到Gitee仓库
• 无需重新部署即可更新内容
